ubuntu20.04安装PX4 Firmware/Autopilot的

2022/11/20 教程 px4

暂未实测（）后面会自己重新跑一遍的.

环境：ubuntu20.04+ros-noetic+gazbeo11

依赖安装

sudo apt install ninja-build exiftool ninja-build protobuf-compiler libeigen3-dev genromfs xmlstarlet libgstreamer1.0-dev libgstreamer-plugins-base1.0-dev gawk

ros安装

# 设置软件库
sudo sh -c 'echo "deb http://packages.ros.org/ros/ubuntu $(lsb_release -sc) main" > /etc/apt/sources.list.d/ros-latest.list'
# 设置KEY
curl -s https://raw.githubusercontent.com/ros/rosdistro/master/ros.asc | sudo apt-key add -
# 更新软件列表
sudo apt update
# 安装完整ROS包
sudo apt install ros-noetic-desktop-full

mavros的安装

# 安装mavros包
sudo apt install ros-noetic-mavros ros-noetic-mavros-extras
# 下载脚本
wget https://gitee.com/robin_shaun/XTDrone/raw/master/sitl_config/mavros/install_geographiclib_datasets.sh
# 添加权限
sudo chmod a+x ./install_geographiclib_datasets.sh
# 执行脚本 安装依赖
sudo ./install_geographiclib_datasets.sh

python

  • 1.装了conda的话跳过此步骤

  • 2.pip -V和python -V正常显示版本则代表已安装好

  • 3.pip -V或python -V显示command not found

    可以用sodu apt install python3 python3-pip安装一下python3

    或者使用源码编译或者使用ubuntu的工具进行管理，自行搜索一下，推荐使用第一种情况提到的conda

PX4配置

# 下载v1.11.3的版本
git clone -b v1.11.3 https://github.com/PX4/PX4-Autopilot.git
# 进入目录
cd PX4-Autopilot
# 子模块更新（因为px4仓库里面放了很多其他的仓库，这步在代理不稳定时极容易失败，反复运行直到没有报错，也可以使用下面的压缩包）
git submodule update --init --recursive
# 编译（极容易报错）
make px4_sitl_default gazebo

基地的同学可以用这个压缩包

TODO：后面测试完毕之后传一份网盘~

编译完成出现gazebo，会出现一架飞机

将下面的内容写入**.bashrc**

source ~/catkin_ws/devel/setup.bash
source ~/PX4-Autopilot/Tools/setup_gazebo.bash ~/PX4-Autopilot/ ~/PX4-Autopilot/build/px4_sitl_default
export ROS_PACKAGE_PATH=$ROS_PACKAGE_PATH:~/PX4-Autopilot
export ROS_PACKAGE_PATH=$ROS_PACKAGE_PATH:~/PX4-Autopilot/Tools/sitl_gazebo

重新打开一个终端会出现几个路径，则设置代表成功

如果不想每次都看见那几行路径，则去~/PX4-Autopilot/Tools/setup_gazebo.bash中把echo的几句话全部注释掉即可

新开一个终端，运行下面的launch

roslaunch px4 mavros_posix_sitl.launch

执行rostopic echo /mavros/state,connected为true则成功

真机与飞控相连时会用到QGroundControl
